# 先声药业集团PPT生成器

## 概述

基于先声药业集团2026版官方PPT模板，根据用户提供的汇报需求，**智能理解意图并自动生成**符合集团规范的PPT文件。生成结果完整继承模板的母版、主题、版式、Logo、页脚、主题色和字体规范。

**架构设计**：底层5大智能能力 × 上层8个业务模块。能力层负责"怎么生成得好"，模块层负责"生成什么内容"。

## 触发条件

当用户出现以下任一表述时触发此Skill：
- "帮我做一份先声药业的PPT"
- "用先声模板生成汇报材料"
- "先声药业项目汇报PPT"
- "Simcere presentation"
- 任何与"先声药业"+"PPT/幻灯片/汇报"相关的请求

# 第一部分：底层能力层（5大智能能力）

这5项能力贯穿所有业务模块，是整个Skill的"操作系统"。

## 能力1：场景识别与模块路由

### 工作原理

从用户的一句话需求中提取4类关键信号，匹配到对应业务模块：

| 信号类型 | 提取内容 | 示例 |
|---------|---------|------|
| 主题词 | 产品名、品牌名、项目名 | 科唯可、英太青、睡眠驿站 |
| 动作词 | 核心意图 | 复盘/规划/汇报/分析/盘点 |
| 时间词 | 时间范围 | H1/H2/Q1/上半年/2026年 |
| 维度词 | 业务领域 | 渠道/产品/品牌/费用/团队/提示物 |

### 模块路由表

| 模块 ID | 业务场景 | 典型触发词 | 默认页数 |
|---------|---------|-----------|---------|
| `review` | 业务复盘 | 复盘、回顾、H1、H2、季度总结、年度回顾、上半年、下半年 | 10-15页 |
| `campaign` | 营销计划 | 营销方案、Campaign、推广计划、品牌传播、上市计划、品牌策略 | 10-18页 |
| `brand-tool` | 品牌提示物 | 提示物、物料、礼品、赠品、品牌周边、推广物品、DA | 6-10页 |
| `channel` | 渠道策略 | 渠道、连锁、药店、终端、覆盖、县域、开发、分销 | 8-15页 |
| `product` | 产品策略 | 产品定位、竞品、差异化、适应症、DORA、科唯可、英太青 | 8-12页 |
| `data-report` | 数据汇报 | 数据、指标、KPI、达成、趋势、销售数据、目标 | 6-12页 |
| `data-insight` | 数据洞察 | 分析这组数据、数据洞察、帮我看看、数据分析、找出规律、异常点 | 6-10页 |
| `org-capability` | 组织能力 | 培训、赋能、团队、架构、能力建设、人才 | 6-10页 |
| `general` | 通用汇报 | 其他未明确归类的汇报需求 | 8-12页 |

**路由规则**：对用户输入做关键词匹配计分。匹配到多个模块时选最高分。无法确定时归入 `general`。

**特殊路由规则**：
- 如果用户**直接提供了数据**（粘贴了表格/数字/截图）且没有明确的业务模块指向，优先路由到 `data-insight`
- 如果用户既提供了数据又指向了业务场景（如"帮我复盘这组覆盖数据"），按业务场景路由（`review`），数据作为输入融入该模块

## 能力2：结构化需求澄清（多轮对话）

### 核心原则

- **渐进收敛**：分3轮逐步澄清，每轮不超过3个问题
- **动态调整**：根据已获取信息跳过不需要的问题
- **数据边界**：涉及数据时明确告知"未提供将使用[待补充]占位，绝不编造"

### 第一轮：场景与范围确认（必做）

用一句话复述理解，确认场景正确：

> 我理解您要做一份关于「{主题}」的{模块名}汇报，面向{推测听众}，重点围绕{推测核心}。这样理解对吗？

如信息不足，追问：
- **目标听众**：集团领导 / 事业部负责人 / 区域经理 / 客户 / 内部团队？
- **汇报时段**：H1复盘 / H2规划 / 全年 / 季度？
- **核心产品/品牌**：涉及哪个产品或品牌线？

### 第二轮：模块专属追问（按需）

根据识别到的模块，追问该模块的专属问题（详见各模块的"追问清单"章节）。

### 第三轮：数据边界确认（必做）

- 对用户未提供的具体数据，明确告知将使用 **"[待补充]"** 占位
- 如用户提供数据，确认数据来源和时间口径
- 涉及金额/费用时，必须确认是否可以展示

## 能力3：智能页面类型推荐

### 推荐决策树

根据每页内容的语义特征，自动选择最佳页面类型，用户无需手动指定：

```
内容特征判断：
├─ 趋势/变化 + 多时间点数据 → chart(折线图)
├─ 占比/构成/分布 + 分类 → chart(饼图) 或 table
├─ 对比/vs/差异 + 多组 → comparison
├─ 时间线 + 多阶段动作 → timeline_horizontal
├─ 月度规划 + 多维度 → gantt 或 calendar_grid
├─ 核心数字 + 需冲击力 → big_number
├─ 多指标 + 卡片式 → kpi_dashboard
├─ 步骤/阶段/流程 → process
├─ 做的好/不好 + 维度分类 → review_matrix
├─ 持续做/优化/开始做 + 分类 → action_category
├─ 架构/体系/多维升级 → strategy_diagram
├─ 物品盘点/场景分类/评分 → item_matrix
├─ 月度排期/资源分配 → calendar_grid
├─ 纯文字要点(3-6条) → content(bullet)
├─ 文字要点 + 数据表格 → content(body+table)
└─ 仅过渡/分隔 → section
```

### 搭配规则

- **封面**：始终为第1页
- **目录**：始终为第2页（页数≤5时可选跳过）
- **章节分隔**：每个章节开始前插入 section 页
- **避免重复**：相邻页面不连续使用同一种类型（尤其避免连续3页content）
- **视觉节奏**：每3-4页内容后插入一个视觉变化（图表/矩阵/时间线）
- **结束页**：始终为最后一页

### 内容量自适应

- body列表：单页≤6条，超过自动分页
- 表格：单页≤8行(含表头)，超过拆分或精简
- timeline_horizontal：≤6个阶段，超过改用gantt
- KPI卡片：单页2-6个，最优3-4个

### 术语体系

生成内容时自动注入以下行业术语，提升专业性：

| 领域 | 术语 |
|------|------|

| 产品/医学 | 适应症、作用机制(MOA)、半衰期、DORA、GABA-A、循证证据、真实世界研究、患者旅程 |

### 叙事风格规范

所有模块生成内容必须遵循：

- **结论先行**：每页标题即结论，正文展开论据
- **数据说话**：能用数字就不用形容词；数字有对比才有意义（如不写"增长显著"，写"同比增长+13.6%"）
- **动作明确**：不写"加强覆盖"，写"新增铺货5000家（目标覆盖率75%）"
- **逻辑闭环**：背景→问题→策略→动作→资源→指标，缺一不可
- **语言务实**：避免"赋能""抓手""闭环""拉通"等空洞词汇，用具体动词和名词
- **药企语境**：保留必要行业术语，但不过度使用

## 能力5：内容质量自动检查

### 5.1 生成前检查（大纲 & JSON阶段）

| 检查项 | 规则 | 不合格处理 |
|--------|------|-----------|
| 逻辑连贯性 | 叙事线：封面→目录→背景→分析→策略→执行→资源→指标→结束 | 调整页面顺序 |
| 页面主题唯一性 | 每页只表达一个核心主题 | 拆分过载页面 |
| 标题即结论 | 每页标题能独立传达该页核心信息 | 改写为结论式标题 |
| 数据一致性 | 同一指标在不同页面数值一致 | 统一数据口径 |
| 篇幅适配 | body≤6条，表格≤8行×5列 | 拆分或精简 |
| 布局多样性 | 相邻页不使用同类型 | 插入过渡页或更换类型 |
| 无空占位 | 所有slide都有实质内容 | 删除空白页 |
| 数据标注 | 所有数据页标注来源或"[待补充]" | 补充标注 |

### 5.2 生成后检查（PPT产出阶段）

| 检查项 | 验证方式 |
|--------|---------|
| 模板母版完整 | 检查Logo、页脚、版权信息 |
| 表格为真实对象 | 非色块拼贴 |
| 图表为真实对象 | 非截图 |
| 字体统一 | 中文微软雅黑、英文Arial |
| 无模板残留文字 | 无"此处可填入小标题"等占位符 |
| 文字不溢出 | 内容在页面边界内 |
| 可编辑性 | 文本框、表格、图表均可选中编辑 |

### 5.3 内容合理性校验

- **数据合理性**：增长率±200%内、市场份额0-100%、复购率0-100%
- **时间逻辑**：H1数据对应上半年，H2规划对应下半年
- **金额量级**：费用金额与行业常识匹配
- **因果关系**：策略→动作→指标间存在可推导的因果链

# 第二部分：业务模块层（9大场景模块）

每个模块包含：叙事框架、页面编排、追问清单、数据框架、专属页面类型偏好。

## 模块1：业务复盘 (`review`)

### 触发条件
用户提及：复盘、回顾、H1、H2、季度总结、年度回顾、上半年、下半年

### 核心逻辑

先声零售业务的复盘遵循**五维业务链路**：传播→覆盖→陈列→培训→复购。这五个维度构成从"让人知道"到"让人持续买"的完整漏斗，每个维度既是复盘维度，也是规划维度。在此基础上叠加**有效动作三分法**（做的好/做的不好/没有做），形成"五维×三类"的复盘矩阵。最终落脚到**投入产出比**，回答老板最关心的问题：钱花得值不值。

### 标准叙事弧线

```
封面
  → 目录
  → 章节分隔：复盘概述
  → 复盘总览：调研方法 × 覆盖范围 (content)
  → 五维业务链路总览 (process)
  → 章节分隔：五维复盘
  → 传播复盘：曝光×传播 (content + table)
  → 覆盖复盘：门店×连锁 (content + table)
  → 陈列复盘：终端可见度 (content + table)
  → 培训复盘：会讲会卖 (content + table)
  → 复购复盘：患者留存 (content + table)
  → 有效动作复盘矩阵 (review_matrix)
  → 章节分隔：反思与改进
  → 做的好 → 持续做 (action_category)
  → 做的不好 → 优化提升 (action_category)
  → 没有做 → 开始做 (action_category 或 strategy_diagram)
  → 章节分隔：投入产出
  → 费用投入复盘：分项费用×ROI (content + table)
  → 整体投入产出比分析 (content + table 或 comparison)
  → 下一步关键计划 (kpi_dashboard)
  → 结束
```

### 页面编排规则

| 页面顺序 | 强制页面类型 | 核心内容 | 可替换类型 |
|---------|------------|---------|-----------|
| 1 | cover | 封面 | - |
| 2 | toc | 目录 | - |
| 3 | section | 复盘概述 | - |
| 4 | content | 调研方法×覆盖范围 | - |
| 5 | process | 五维链路总览：传播→覆盖→陈列→培训→复购 | content |
| 6 | section | 五维复盘 | - |
| 7 | content + table | 传播复盘 | content + chart |
| 8 | content + table | 覆盖复盘 | content + chart |
| 9 | content + table | 陈列复盘 | content + chart |
| 10 | content + table | 培训复盘 | content + chart |
| 11 | content + table | 复购复盘 | content + chart |
| 12 | review_matrix | 有效动作复盘矩阵 | - |
| 13 | section | 反思与改进 | - |
| 14 | action_category | 做的好→持续做 | - |
| 15 | action_category | 做的不好→优化提升 | - |
| 16 | action_category/strategy_diagram | 没有做→开始做 | - |
| 17 | section | 投入产出 | - |

> **页面裁剪规则**：如果复盘仅覆盖部分维度（如只复盘"覆盖+培训"），则删减对应页面，最低保留：封面+目录+复盘概述+2个维度复盘+review_matrix+1个action_category+费用复盘+投入产出+指标+结束 ≈ 12页

### 追问清单

1. 复盘覆盖哪些维度？（传播/覆盖/陈列/培训/复购——全部还是部分？）
2. 是否有调研数据支撑？（高管访谈/门店走访/问卷/流向数据）
3. 核心结论方向？（哪些做的好？哪些待改进？哪些没做？）
4. 是否需要费用拆解和投入产出分析？
5. 是否有去年同期数据做对比？

### 五维业务链路指标体系

这是先声零售业务的核心漏斗模型，每个维度的指标必须可量化、可追踪：

| 维度 | 核心问题 | 关键指标 | 数据来源 |
|------|---------|---------|---------|
| **传播** | 让人知道了吗？ | 线上场次、线下场次（科普/义诊）、品牌曝光人次、单场覆盖成本、媒体触达量 | 品牌部数据、活动系统 |
| **覆盖** | 铺到店了吗？ | 门店覆盖数、连锁开发度（TOP100/300）、铺货率、新增门店数、县域覆盖数 | 流向数据、SFE系统 |
| **陈列** | 看得见吗？ | 陈列门店数、陈列合格率、POSM到位率、专柜/端架占比 | 门店走访、代表汇报 |
| **培训** | 会讲会卖吗？ | 培训场次、覆盖店员/药师人数、考核通过率、产品知识掌握度 | 培训系统、考核数据 |
| **复购** | 会回来买吗？ | 复购率、会员转化率、疗程用药率、患者留存率、NPS | 流向数据、CRM系统 |

**链路逻辑**：传播→覆盖→陈列→培训→复购，前一步是后一步的前提，任何一步断裂都会影响最终产出。

### 有效动作三分法

在每个维度下，将动作分为三类：

| 分类 | 含义 | 页面类型 | 典型表述 |
|------|------|---------|---------|
| 做的好 → **持续做** | 有数据验证的有效动作，需坚持和深化 | action_category | "321世界睡眠日整合营销，品牌曝光超500万人次" |
| 做的不好 → **优化提升** | 方向正确但执行有差距，需改进 | action_category | "1-4月科普义诊2100场，线上仅135场，覆盖效率待提升" |
| 没有做 → **开始做** | 应该做但未启动的，需规划 | action_category / strategy_diagram | "优质睡眠健康驿站：陈列升级+客流升级" |

### 投入产出分析框架

老板最关心的三件事：**花了多少钱、花在哪了、值不值**。

```
费用结构（花在哪了）：
├─ 传播费用：占比%、金额、主要用途
├─ 覆盖费用：占比%、金额、主要用途
├─ 陈列费用：占比%、金额、主要用途
├─ 培训费用：占比%、金额、主要用途
├─ 复购/患者管理费用：占比%、金额、主要用途
└─ 提示物费用：占比%、金额、主要用途

产出衡量（值不值）：
├─ 整体ROI = 产出销售额 / 总投入费用
├─ 分维度ROI：每个维度的投入产出比
├─ 费用使用率 = 实际花费 / 预算
├─ 人效 = 销售额 / 代表人数
└─ 单店产出 = 销售额 / 覆盖门店数

对比基准：
├─ 同比：vs 去年同期
├─ 环比：vs 上一周期
└─ 预算比：实际 vs 预算
```

### 数据来源标注规范
- 高管访谈：标注"高管访谈（N人）"
- 门店走访：标注"门店走访（N家/月）"
- 问卷调研：标注"一线问卷（样本量N）"
- 流向数据：标注"流向系统数据"
- SFE数据：标注"SFE系统数据"
- 财务数据：标注"财务系统数据"

## 模块2：营销计划 (`campaign`)

### 触发条件
用户提及：营销方案、Campaign、推广计划、品牌传播、上市计划、品牌策略、市场活动

### 标准叙事弧线

```
封面
  → 目录
  → 章节分隔：市场背景
  → 市场洞察与机会 (content + chart)
  → 竞品格局分析 (comparison 或 content + table)
  → 章节分隔：核心策略
```

### 五维数据汇报指标体系

每个维度不是孤立的数字，而是**五维链路中的一环**——传播贡献覆盖率，覆盖支撑陈列率，陈列+培训驱动动销，培训+复购决定留存。数据汇报时要呈现这种链路关系。

| 维度 | 汇报指标 | 对比基准 | 链路逻辑 |
|------|---------|---------|---------|
| **传播** | 线上/线下场次、曝光人次、单场成本、品牌认知度 | 同比、预算比 | 传播→驱动覆盖：品牌活动为覆盖开路 |
| **覆盖** | 门店数、连锁开发度、铺货率、新增门店、县域渗透 | 同比、目标达成率 | 覆盖→支撑陈列：铺到店才能陈列 |
| **陈列** | 陈列门店数、陈列合格率、POSM到位率、专柜占比 | 环比、标准线 | 陈列→承接培训：看见才能讲清 |
| **培训** | 培训场次、覆盖人数、考核通过率、产品知识掌握度 | 同比、目标达成率 | 培训→驱动复购：会讲会卖才复购 |
| **复购** | 复购率、会员转化率、疗程用药率、患者留存率 | 同比、环比 | 复购→最终产出：留存即增长 |

### 投入产出分析框架

这是数据汇报的**核心收口**，所有维度的数据最终汇聚到"花了多少钱，产出了多少"：

```
三层投入产出分析：

第一层：整体投入产出
├─ 总投入（费用合计）vs 总产出（销售额/纯销）
├─ 整体ROI = 产出 / 投入
├─ 费用率 = 总费用 / 销售额
└─ 同比变化：ROI是否改善

第二层：分维度投入产出
├─ 传播费用 vs 传播产出（曝光量/覆盖增量）
├─ 覆盖费用 vs 覆盖产出（新增门店数/铺货率提升）
├─ 陈列费用 vs 陈列产出（陈列合格率/终端可见度）
├─ 培训费用 vs 培训产出（店员掌握度/推荐率）
├─ 复购费用 vs 复购产出（复购率/会员转化率）
└─ 找出：哪个维度投入产出比最高？哪个最低？

第三层：效率指标
├─ 人效 = 销售额 / 代表人数
├─ 单店产出 = 销售额 / 覆盖门店数
├─ 单场活动成本 = 传播费用 / 活动场次
├─ 单店培训成本 = 培训费用 / 培训门店数
└─ 费使用率 = 实际花费 / 预算额度
```

### 数据口径规范

| 术语 | 定义 | 注意事项 |
|------|------|---------|
| 纯销 | 终端实际售出金额 | 区别于"发货"，是真正动销 |
| 发货 | 工厂/经销商出库金额 | 不等于终端销售 |
| 流向数据 | 药品从流通到终端的数据 | 反映实际覆盖和动销 |
| 连锁开发度 | 已合作连锁数 / 目标连锁数 | 分TOP100/TOP300统计 |
| 铺货率 | 有货门店数 / 总门店数 | 区分首铺和持续供货 |
| 动销率 | 有销售记录门店 / 有货门店 | 反映终端真实销售能力 |
| 复购率 | 二次购买患者 / 首次购买患者 | 区分3个月/6个月口径 |
| ROI | 产出金额 / 投入金额 | 统一口径：纯销/费用 |

### 指标卡片标准格式

```
每个指标含：
- 指标名称（如"纯销金额"）
- 当前值 + 单位（如"14.2亿元"）
- 同比变化（如"+13.6%"）
- 环比变化（如"+3.2%"）
- 目标值（如"15亿元"）
- 达成率（如"94.7%"）
- 趋势箭头（↑→↓）
```

### 五步分析法详解

#### 第一步：结构化梳理

将用户提供的原始数据（可能是零散的数字、表格截图、一段文字描述）转化为结构化格式：

```
输入：用户粘贴的原始数据
处理：
  ├─ 识别数据维度（时间/区域/产品/渠道/指标）
  ├─ 识别数值类型（绝对值/比率/金额/数量）
  ├─ 识别时间跨度（单点/月度/季度/年度）
  └─ 识别可对比项（有对比基准 vs 无对比基准）
输出：结构化数据框架
```

#### 第二步：异常/趋势识别

对结构化数据执行四类分析：

| 分析类型 | 方法 | 输出形式 |
|---------|------|---------|
| **趋势** | 时间序列走向，上升/下降/平稳/拐点 | chart(折线图) + 文字结论 |
| **异常** | 偏离均值/趋势的数据点，异常高/异常低 | comparison + 文字标注 |
| **结构** | 占比、构成、集中度，谁是主力/谁是短板 | chart(饼图) 或 table + 文字结论 |
| **关联** | 维度间的相关性，A涨B是否也涨 | content + 文字结论 |

**判断优先级**：异常 > 趋势拐点 > 结构变化 > 一般趋势

#### 第三步：根因假设

对发现的异常和趋势，提出**可验证的根因假设**（不是结论，是假设）：

```
发现：Q2覆盖门店数环比下降15%
假设1：某大连锁合作终止 → 验证方式：检查连锁维度数据
假设2：代表离职导致区域空缺 → 验证方式：检查区域维度数据
假设3：季节性波动（去年Q2也下降？）→ 验证方式：同比对比
```

#### 第四步：行动建议

基于洞察，给出**具体的、可执行的建议**：

| 洞察类型 | 行动建议模板 |
|---------|------------|
| 异常增长 | "X指标异常增长Y%，建议：①确认数据口径 ②若属实，分析驱动因素 ③判断是否可持续" |
| 异常下降 | "X指标异常下降Y%，建议：①排查数据完整性 ②定位下降最大维度 ③评估影响范围" |
| 结构失衡 | "X占比过高(Y%)，Y占比偏低(Z%)，建议：①评估是否需要再平衡 ②确定优先强化维度" |
| 趋势拐点 | "X指标在M月出现拐点，建议：①回溯M月关键事件 ②判断拐点是否持续 ③调整规划" |

#### 第五步：PPT呈现

将分析结果按叙事弧线编排为PPT页面，遵循以下规则：

- **结论先行**：每个chart/table页的标题即核心结论（如"Q2覆盖门店环比下降15%，主要由华东区域拖动"）
- **数据+洞察成对出现**：不单独放一个图表，图表后面必须跟一句洞察
- **标注数据来源**：每页数据标注"数据来源：{来源}，时间范围：{范围}"
- **区分事实与假设**：事实用陈述句，假设用"可能因为…"，建议用"建议…"

### 数据输入适配

用户可能以多种方式提供数据，AI需要灵活适配：

| 输入方式 | 处理方法 |
|---------|---------|
| 粘贴数字/表格 | 直接解析，转化为结构化数据 |
| Excel/CSV文件 | 读取文件，自动识别维度和指标 |
| 截图中的数据 | OCR识别或手动提取，确认后使用 |
| 口头描述 | "Q1销售额12.5亿，Q2增长到14.2亿" → 结构化 |
| 文档中的数据 | 从文档中提取关键数字，确认后使用 |

> **关键原则**：无论哪种输入方式，必须在PPT的"数据概览"页明确标注数据来源和时间口径，确保可追溯。

## 各步骤详解

### Step 1: 场景识别
调用**能力1**。从用户输入中提取关键信号，匹配业务模块。输出识别结果让用户确认。

### Step 2: 需求澄清
调用**能力2**。按3轮渐进式追问，结合当前模块的专属追问清单。

### Step 3: 大纲生成
结合**当前模块的叙事弧线 + 能力3的页面推荐 + 能力4的行业知识**，生成结构化大纲：

```markdown
## PPT大纲预览

**标题**: {主标题}
**副标题**: {副标题}
**模块**: {模块名} | **页数**: {N}页 | **听众**: {听众}

| # | 页面类型 | 页面标题 | 核心内容（一行概括） |
|---|---------|---------|-------------------|
| 1 | cover | {标题} | 封面：{副标题} / {日期} / {汇报人} |
| 2 | toc | 目录 | {章节列表} |
| ... | ... | ... | ... |
| N | ending | 谢谢 | Thank You |
```

**必须等待用户确认大纲后再继续。**

### Step 4: 内容生成与JSON构建
调用**能力3+4**。根据确认的大纲，逐页生成完整内容：

**内容生成优先级**：
1. 用户明确提供的数据 → 直接使用
2. 用户提供方向但无具体数据 → 行业合理框架 + "[待补充]"标注
3. 用户完全未提供 → 行业通用框架 + "[待补充]"标注

**JSON构建规则**：
- 未提供的数据用"[待补充]"占位，**不可编造**
- body数组：`{"text": "...", "level": 0}` 格式，level为缩进层级
- table：headers + rows二维数组
- chart：chart_type、categories、series

### Step 5: PPT生成

```bash
python scripts/generate.py \
  --template assets/simcere-template.pptx \
  --input /tmp/content.json \
  --output /workspace/output.pptx
```

### Step 6: 质量验证
调用**能力5**的生成后检查清单，逐项验证。

### Step 7: 交付
1. .pptx文件链接
2. 逐页内容摘要
3. "[待补充]"数据标注清单（如有）
4. 询问是否需要导出PDF

# 第五部分：设计规范

## 视觉规范

### 颜色
- **主色**: accent1 #00B052 (绿色) - 表头、强调、重点标识
- **辅助色**: accent3 #006647 (深绿) - 二级强调
- **文字色**: dk1 #333333 (深灰) - 正文
- **背景色**: lt1 #FFFFFF (白色) - 页面背景
- **避免**: 高饱和、花哨、互联网感配色

### 字体
- 中文：**微软雅黑**（全局统一）
- 英文：**Arial**（全局统一）

### 字号层级
| 元素 | 字号 | 样式 |
|------|------|------|
| 封面大标题 | 36pt | 粗体 |
| 页面标题 | 28-32pt | 粗体 |
| 章节标题 | 32pt | 粗体 |
| 副标题 | 14-16pt | 常规 |
| 正文 | 14pt | 常规 |
| 表格文字 | 11-12pt | 常规 |

### 表格规范
- 必须生成真正的PPT表格对象（非色块拼贴）
- 表头：绿色背景(#00B052) + 白色粗体文字(12pt)
- 数据行：11pt，隔行浅绿背景(#F0F7F4)

### 图表规范
- 真实图表对象，非截图
- 配色顺序：绿→亮绿→深绿→青→黄→橙
- 标题、图例、坐标轴、数据标签完整

### 排版规范
- 每页一个核心主题
- 标题明确，正文分层
- 重点突出但不过度装饰
- 留白充足，文字不贴边
- 严格对齐

## 禁止事项

- 不得自行重画先声Logo，必须使用模板内置Logo
- 不得使用非模板体系的花哨配色
- 不得把表格做成色块拼贴
- 不得输出不可编辑的整页图片
- 不得改变模板的页脚、页码、版权信息
- 不得随意更改字体体系
- 不得只模仿颜色，忽略字号、行距、母版、参考线
- 不得生成AI感、海报感、营销长图感的页面
- **不得编造数据**：未提供的数据使用"[待补充]"占位
