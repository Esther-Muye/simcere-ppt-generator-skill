# 先声药业集团PPT生成器

## 概述

基于先声药业集团2026版官方PPT模板，根据用户提供的汇报思路和内容框架，自动生成可直接使用的PPT文件。生成结果完整继承模板的母版、主题、版式、Logo、页脚、主题色和字体规范。

## 触发条件

当用户出现以下任一表述时触发此Skill：
- "帮我做一份先声药业的PPT"
- "用先声模板生成汇报材料"
- "先声药业项目汇报PPT"
- "Simcere presentation"
- 任何与"先声药业"+"PPT/幻灯片/汇报"相关的请求

## 工作流程

### Step 1: 收集信息

向用户确认以下信息（如用户未提供）：

| 信息项 | 说明 |
|--------|------|
| PPT主题 | 汇报的核心主题 |
| 使用场景 | 领导汇报/客户沟通/内部复盘/项目启动会/策略规划等 |
| 目标听众 | 向谁汇报（如：集团领导、区域经理、客户等） |
| 页数要求 | 期望的幻灯片数量（如未指定，默认8-12页） |
| 核心内容 | 需要涵盖的核心议题、章节、要点 |
| 希望强调的结论 | 最关键的结论或行动号召 |
| 是否需要表格 | 数据对比、进度跟踪等 |
| 是否需要图表 | 趋势、占比、矩阵等数据可视化 |
| 是否需要时间轴 | 项目节奏、里程碑规划 |
| 是否需要机制图 | 组织架构、流程机制、责任分工 |

### Step 2: 规划PPT结构

**⚠️ 关键：必须遵循标准叙事框架**

基于用户提供的信息，规划页面结构。页面类型包括：

| 页面类型 | 用途 |
|----------|------|
| cover | 封面页，含标题、副标题、日期、汇报人 |
| toc / 目录 | 目录页，列出章节结构 |
| section / 节标题 | 章节分隔页 |
| content / bullet | 标准内容页，含标题+正文列表+可选表格 |
| two-column | 双栏对比布局 |
| table | 数据表格页 |
| chart | 图表页（柱状图/折线图/饼图等） |
| timeline | 时间轴/里程碑页 |
| quote | 引用/金句页 |
| image | 图片展示页 |
| ending / 结束 | 结束页，"谢谢" |

**标准叙事框架（强制遵循）：**

```
封面 → 目录/议程 → 复盘（市场调研×费用投入）
  → 反思分类:
       ├─ 有效动作 × 做的好 → 持续做 (action_category)
       ├─ 有效动作 × 做的不好 → 优化提升 (action_category)
       └─ 有效动作 × 没有做 → 开始做 (action_category / strategy_diagram)
  → 费用投入规划 → 结束
```

**页面结构建议（根据内容自动选择）：**

```
封面 → 目录 → 项目背景/现状 → 核心结论 → 
策略框架 → 行动路径 → 时间节奏 → 资源支持 → 
数据复盘/表格 → 问题诊断 → 下一步计划 → 总结/结束
```

**内容逻辑检查清单（生成前必须确认）：**
1. ✅ 是否遵循标准叙事框架（复盘→反思→规划）
2. ✅ 每页是否只表达一个核心主题
3. ✅ 标题是否明确，正文是否分层
4. ✅ 重点信息是否突出但不过度装饰
5. ✅ 相邻页面是否避免重复同一种布局类型

### Step 4: 生成PPT

使用 Python 生成脚本创建PPT：

```bash
python ~/.workbuddy/skills/simcere-ppt/scripts/generate.py \
  --template ~/.workbuddy/skills/simcere-ppt/assets/simcere-template.pptx \
  --input /tmp/content.json \
  --output /tmp/output.pptx
```

### Step 5: 质量检查

生成后必须进行以下检查：

1. **内容检查**
   - 确认无"待补充"或"示例占位"残留在最终版本（如果用户要求）
   - 确认页码连贯
   - 确认无错别字

2. **格式检查**
   - 确认每页只有一个核心主题
   - 确认标题层级清晰
   - 确认文字未溢出或贴边
   - 确认表格为真实表格对象（非色块拼贴）

3. **风格检查**
   - 确认使用先声主题色（绿色#00B052为主）
   - 确认使用中文字体微软雅黑、英文字体Arial
   - 确认Logo未丢失或被替换
   - 确认页脚"Copyright © 2026 Simcere. All rights reserved."完整

4. **可编辑性检查**
   - 确认文本框可编辑
   - 确认表格可编辑
   - 确认无整页截图

## 版式使用指南

### 封面 (cover)
- 使用模板原"封面"版式
- 包含：Logo、主标题、副标题、日期、汇报人
- 适用：PPT首页

### 目录 (toc)
- 使用模板原"1_仅logo页"版式
- 适用：汇报大纲概览

### 章节分隔 (section)
- 使用模板原"节标题"版式
- 包含：章节标题、副标题
- 适用：各大章节之间的过渡页

### 标准内容 (content)
- 使用模板原"标题和内容"版式
- 包含：标题 + 项目符号列表正文，可嵌入表格或图表
- 适用：大多数内容页
- 参数：`body`（项目符号列表）、`table`（表格数据）、`chart`（图表数据）

### 甘特图 (gantt)
- 矩阵式甘特图：纵轴为动作类别，横轴为月份/时间
- 包含：绿色表头 + 色块单元格 + 底部里程碑标签 + 结论
- 适用：项目推进节奏、月度行动计划
- 参数：`rows`（纵轴标签）、`months`（横轴标签）、`data`（高亮单元格）、`milestones`、`conclusion`

### 横向时间线 (timeline_horizontal)
- 按月/阶段横向排列的卡片式时间线
- 包含：月份标签（绿色背景）+ 主题 + 目标人群 + 具体动作列表
- 适用：Campaign规划、品牌传播节奏
- 参数：`items`（每个阶段卡片）、`subtitle`（逻辑主线说明）

### 大数字展示 (big_number)
- 超大字号核心数字 + 说明文字 + 底部成本/数据卡片
- 包含：大数字区域 + 右侧信息卡片 + 底部明细卡片
- 适用：活动规模展示（如"5,000场科普"）、成本拆解
- 参数：`big_number`、`unit`、`description`、`side_cards`、`cards`

### 对比页 (comparison)
- 多组数据并排对比卡片
- 包含：顶部引言 + 对比卡片（标签+大数值+说明）+ 底部结论
- 适用：竞品对比、数据对比（如复购率对比）
- 参数：`intro`、`items`、`target`、`note`

### 流程/阶段 (process)
- 水平排列的圆角矩形节点 + 箭头连接
- 包含：阶段标题 + 描述 + 箭头
- 适用：里程碑、上市节奏、三年规划
- 参数：`steps`（步骤列表）、`direction`（horizontal/vertical）

### KPI指标卡 (kpi_dashboard)
- 网格排列的指标卡片
- 包含：标签 + 大数值 + 变化率/目标
- 适用：关键指标总览、业务目标展示
- 参数：`kpis`（指标列表，每个含label/value/unit/change/target）

### 物品矩阵图 (item_matrix)
- 多列矩阵布局：场景 → 对象 → 物品 → 成本分档 → 评分
- 包含：左侧场景色块（跨行合并视觉效果）+ 对象列 + 物品列表列 + 成本分档列 + 平均喜爱度列
- 适用：品牌提示物盘点、资源矩阵、产品组合展示、成本-满意度矩阵
- 参数：
  - `groups`（分组列表，每个含 scene/audience/cost_tier/avg_score/items）
  - `items`（每个物品含 name/score）
  - `footer_notes`（底部说明，可选）
- 注意：每个 group 的 items 数量决定该行高度，物品越多行越高

### 日历网格 (calendar_grid)
- 行×列网格布局：行=场景/维度，列=月份/阶段
- 包含：顶部月份绿色表头 + 左侧场景深绿行标题 + 网格单元格（物品列表）
- 适用：月度推广排期、物品使用规划、项目时间网格、资源分配矩阵
- 参数：
  - `rows`（行标题列表，每个含 name/subtitle）
  - `months`（列标题列表）
  - `grid`（二维数据，grid[row_idx][col_idx] = {items: [...], highlight: bool}）
  - `note`（底部备注，可选）
- 注意：每个单元格的 items 为字符串数组，最多显示5个

### 复盘矩阵 (review_matrix)
- 三列矩阵布局：有效动作 | 做的好 | 做得不好
- 左侧深绿动作列 + 中间浅绿"做的好"列 + 右侧浅橙"做得不好"列
- 适用：项目复盘、工作回顾、按维度分优劣的对比分析
- 参数：
  - `badge`（角标，如"复盘"）
  - `category_header`（动作列标题，默认"有效动作"）
  - `good_header`（"做的好"列标题）
  - `bad_header`（"做得不好"列标题）
  - `rows`（行数据，每个含 action/note/good/bad）
  - `footer_note`（底部数据来源说明，可选）
- 注意：行高度根据可用空间自适应，文字内容过长时自动换行

### 分类复盘页 (action_category)
- 左侧色块标签 + 右侧项目符号列表的分类布局
- 适用于"做的好→持续做"、"做的不好→优化提升"、"没有做→开始做"三种复盘场景
- 参数：
  - `badge`（角标，如"反思与改进"）
  - `intro`（页面说明文字，可选）
  - `categories`（分类列表，每个含 label/items）
  - `highlight_box`（底部高亮提示框，含 title/text，可选）
- 注意：分类标签颜色自动循环（深绿/主绿/亮绿），每个分类的项目符号带绿色圆点

### 策略架构图 (strategy_diagram)
- 中心节点 + 多维度箭头连接 + 子节点圆角矩形 + 底部支撑策略
- 适用于：多维升级策略、系统架构、项目推进路线图
- 参数：
  - `badge`（角标，可选）
  - `diagram.center`（中心节点，含 text/sub）
  - `diagram.dimensions`（维度列表，每个含 label/items/sub_label）
  - `diagram.pillars`（底部支撑策略，每个含 label/sub，可选）
- 注意：维度通过Chevron箭头形状连接，子节点为圆角矩形自动排列

### 图表 (chart)
- 作为 content 页面的子元素嵌入
- 支持类型：柱状图(column)、条形图(bar)、折线图(line)、饼图(pie)、面积图(area)
- 适用：趋势、占比、对比等数据可视化
- 参数：`chart_type`、`categories`、`series`、`title`

### 结束页 (ending)
- 使用模板原"末尾幻灯片"版式
- 包含："谢谢"文字、英文副标题
- 适用：PPT最后一页

## 设计规范

### 颜色使用
- **主色**: accent1 #00B052 (绿色) - 用于表头、强调、重点标识
- **辅助色**: accent3 #006647 (深绿) - 用于二级强调
- **文字色**: dk1 #333333 (深灰) - 用于正文
- **背景色**: lt1 #FFFFFF (白色) - 页面背景
- **避免**: 高饱和、花哨、互联网感过强的配色

### 字体使用
- 中文字体必须统一使用 **微软雅黑**
- 英文字体必须统一使用 **Arial**
- 不得出现字号忽大忽小

### 表格规范
- 必须生成真正的PPT表格对象
- 表头绿色背景+白色粗体文字
- 数据行11pt，隔行浅绿背景
- 不得用色块拼贴假表格

### 图表规范
- 优先使用真实图表对象
- 配色遵循主题色顺序：绿→亮绿→深绿→青→黄→橙
- 标题、图例、坐标轴、数据标签完整

### 排版规范
- 每页一个核心主题
- 标题明确，正文分层
- 重点信息突出但不过度装饰
- 页面留白充足，文字不贴边
- 所有元素严格对齐

## 内容逻辑结构

### 标准复盘叙事框架

参考先声药业董事长汇报的标准逻辑结构，推荐按以下弧线组织叙事：

```
封面 → 目录/议程 → 复盘（市场调研×费用投入）
  → 反思分类:
      ├─ 有效动作 × 做的好 → 持续做 (action_category)
      ├─ 有效动作 × 做的不好 → 优化提升 (action_category)
      └─ 有效动作 × 没有做 → 开始做 (action_category / strategy_diagram)
  → 费用投入规划 → 结束
```

### 关键设计原则

- **项目符号**：正文中的列表项使用"•"前缀 + 10-11pt字号
- **不要纯文字页**：避免标题+项目符号的枯燥页面，混入表格/图表/色块/卡片
- **角标**：复盘页左侧添加角标（如"复盘"/"反思与改进"），帮助观众快速定位
- **视觉变化**：相邻页面不要重复同一种布局类型
